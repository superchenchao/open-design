先从一个时间点说起——2026 年 4 月，Claude Design 发布。

大家看左边这张卡片，它证明了什么？LLM 能真正做设计——不是写一篇关于设计的文章，而是直接产出可用的设计稿。这是质变。

但中间这张卡片是问题所在：它闭源、付费、只跑在云端。模型被锁死在 Anthropic 一家，你想换个 Agent 不行，想自部署不行，想用自己的 API Key——也就是 BYOK——还是不行。能力很强，但你拿不到手。

所以才有了右边这张卡片——Open Design 的使命就是把这套能力放开：开源、跑在本地、Agent 随便换。一句话：让最前沿的 AI 设计能力，回到每一个创作者的桌上。

// part 02 · workflow

从想法到交付，四步完成

①
选择起点

一句话描述目标，或从模板 / Plugin 直接选起点。

②
确定视觉方向

选定方向后，色板、字体、间距自动带入生成流程。

③
生成 Artifact

Agent 读取全部上下文，产出真实可运行的文件，沙盒内即时预览和修改。

④
交付 + 记忆沉淀

导出给工程继续开发，或用 HyperFrames 转为视频；每次选择都成为下次生成的上下文。

Open Design 的工作流就四步。我按顺序过一遍。

第一步，选择起点。一句话描述你要什么，或者直接从模板和 Plugin 里挑一个起点——比如今天这份 deck，起点就是一个演讲者模式的 Slides Plugin。

第二步，确定视觉方向。方向一旦选定，色板、字体、间距会自动带进后面的生成流程，不用每页手调。

第三步，生成 Artifact——这是最关键的一步。Agent 读取全部上下文，产出的是真实可运行的文件，不是截图也不是预览图，在沙盒里可以即时预览和修改。

第四步，交付加记忆沉淀。文件可以直接交给工程继续开发，也可以用 HyperFrames 转成营销视频；同时你的每次选择都沉淀为记忆，下次生成更接近你要的结果。

那支撑这套工作流的理念是什么？看下一部分。

// part 03 · principles

Open Design 的三大主张

01
🖥️ 桌面端原生

设计在桌面端发生。本地文件、Figma 导出、代码仓库直接可读，Agent 拥有终端执行全部能力。

02
🤖 不造 Agent，接入 Agent

你电脑上的 Claude Code / Codex / Cursor 已经够强。Open Design 把它们接进完整设计工作流。

03
🧠 越用越懂你

每次选择都沉淀为 Design System、偏好和记忆，下次生成更接近你要的结果。

进入最核心的一部分——Open Design 的三大主张。

第一条，桌面端原生。设计真正发生的地方是桌面：本地文件、Figma 导出、代码仓库，全部直接可读。更重要的是，Agent 在这里拥有终端执行的全部能力——这是云端工具给不了的。

第二条，不造 Agent，接入 Agent。我们不重新发明轮子。你电脑上的 Claude Code、Codex、Cursor 已经够强了，Open Design 做的是把它们接进完整的设计工作流——目前已经接入 21 个 Coding Agent，自动检测 PATH 上的 CLI，零配置。

第三条，越用越懂你。每一次选择都沉淀为 Design System、偏好和记忆，下次生成会更接近你要的结果。这就是自进化的设计系统。

三条合起来：开放、本地、Agent 原生。最后看怎么上手。
