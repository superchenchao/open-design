三档流向，三种风险

L1 · 绿色

本地优先

Open Design：设计稿、记忆、SQLite 全在本地 .od/，BYOK 直连供应商。
风险：自己掌控

L2 · 琥珀

部分上云

云端 SaaS 设计工具：文件存在服务商，导出格式受限，停服即失联。
风险：受制于人

L3 · 红色

全量上云

闭源云端 AI 设计：提示词、产物、Key 全经第三方，模型锁死单家。
风险：不可审计

API Key 绝不离开你的机器

BYOK 模式下，Key 只存在本地 .od/media-config.json，请求直连模型供应商，不经任何中间服务器。

# .od/ · Open Design 本地数据目录 · 全部留在你的机器
local_data:
  - app.sqlite          # 会话、记忆、偏好
  - projects/<id>/      # Agent 工作目录
  - artifacts/          # 保存的设计产物
  - media-config.json   # API Key · 仅本地

byok_direct:
  - providers: [anthropic, openai, azure, google]
    route: 本机 → 供应商，无中间服务器

never_uploaded:
  - "你的提示词"
  - "你的设计稿"
  - "你的 API Key"

选工具前 必过 7 道题

✓
设计稿存在哪？能不能 git diff？（Open Design：本地文件系统）
✓
API Key 经不经过第三方服务器？（BYOK：只存本地，直连供应商）
!
云端工具：提示词和产物会不会被拿去训练？多数条款说不清。
✓
断网或服务商停服，历史产物还能不能打开？（本地文件：永远能）
!
云端工具：模型锁死单家，换 Agent、自部署、BYOK 全做不到。
✓
代码能不能 逐行 审计？（Apache-2.0 全开源，Fork、自部署、商用全允许）
!
云端工具：偏好与记忆数据能不能导出、能不能彻底删除？
